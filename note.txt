* 기본형 VS 참조형
기본형 : int, long, 처럼 변수에 사용할 값을 직접 넣울 수 있는 타입
-> 실제 사용하는 값을 변수에 담을 수 있음
-> 실제 사용하는 값으로 인해서 직접 계산할 수 있다
-> 변수를 선언할때 모두 소문자로 시작한다.
-> java가 기본적으로 제공하는 타입
참조형 : 직접사용하는 값이 아닌, 위치(참조, 주소) 참조값이 들어가 있음
-> 객체와 배열이 있음
-> 참조값으로 인해 직접 계산할 수 없음 but .을 사용해서 객체의 기본 변수로 접근할 경우 계산이 가능해

** 참고 String == class
=> 참조형 이지만 문자형은 매우 자주 다뤄지기 때문에 java에서 편의 기능으로 제공하는것

** 대원칙 : java는 항상 변수의 값을 복사해서 대입한다.

* 기본형과 참조형의 메서드 호출
자바는 메서드의 매개변수(파라미터)는 항상 값이 전달된다. 그러나다 이 값이 실제 값이냐, 참조 값이냐에 따라 달라진다.
기본형 -> 값이 복서되어 전달되기 때문에 매개변수의 값이 변경해도 호출자의 변수값에는 영향이 없다
참조형 -> 참조값이 복사되어 전달될 경우에는 동일한 주소값이 전달이 되어 값이 바뀌게 되면 그 영향을 미치고 있다.

*null
참조형 변수에는 항상 객체가 있는 취이를 가리키는 참조값이 들어간다. 근데 들어갈 위치가 아직 없거나 가리키는 대상을 나중에 입력하고 싶을때 'null'로 표현할 수 있다.

*GC - 아무도 참조하지 않는 인스턴스의 최후
-> NullMain1파일 확인
-> 아무도 참조하지 않는 인스턴스에 관련해서 java가 스스로 삭제를 도와준다.

*NullPointerException
택배를 보내는데 주소지를 없이 보낼 경우!! == 참조값없이 객체를 보낼 경우
null을 가리킬때 발생하는 예외 상황!!

-> 데이터와 기능이 분리되어 있다는 점
ex) 'MusicPlayerData' 와 MusicPlayerMain3에 각각 메서드(기능) 으로 분리되어 있음
이후에 MusicPlayerData가 분리가 되어서 이게 변경이 되면 MusicPlayerMain3에의 메서드 들도 변경 해줘야 한다.
만약 MusicPlayerData.volume을 volume2로 변경시 MusicPlayerMain3에서 MusicPlayerData의 volume을 바라보고 있는 메서드에 영향을 미친다.
