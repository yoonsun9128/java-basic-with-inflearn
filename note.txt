* 기본형 VS 참조형
기본형 : int, long, 처럼 변수에 사용할 값을 직접 넣울 수 있는 타입
-> 실제 사용하는 값을 변수에 담을 수 있음
-> 실제 사용하는 값으로 인해서 직접 계산할 수 있다
-> 변수를 선언할때 모두 소문자로 시작한다.
-> java가 기본적으로 제공하는 타입
참조형 : 직접사용하는 값이 아닌, 위치(참조, 주소) 참조값이 들어가 있음
-> 객체와 배열이 있음
-> 참조값으로 인해 직접 계산할 수 없음 but .을 사용해서 객체의 기본 변수로 접근할 경우 계산이 가능해

** 참고 String == class
=> 참조형 이지만 문자형은 매우 자주 다뤄지기 때문에 java에서 편의 기능으로 제공하는것

** 대원칙 : java는 항상 변수의 값을 복사해서 대입한다.

* 기본형과 참조형의 메서드 호출
자바는 메서드의 매개변수(파라미터)는 항상 값이 전달된다. 그러나다 이 값이 실제 값이냐, 참조 값이냐에 따라 달라진다.
기본형 -> 값이 복서되어 전달되기 때문에 매개변수의 값이 변경해도 호출자의 변수값에는 영향이 없다
참조형 -> 참조값이 복사되어 전달될 경우에는 동일한 주소값이 전달이 되어 값이 바뀌게 되면 그 영향을 미치고 있다.

*null
참조형 변수에는 항상 객체가 있는 취이를 가리키는 참조값이 들어간다. 근데 들어갈 위치가 아직 없거나 가리키는 대상을 나중에 입력하고 싶을때 'null'로 표현할 수 있다.

-> 아무도 참조하지 않는 인스턴스에 관련해서 java가 스스로 삭제를 도와준다.

*NullPointerException
택배를 보내는데 주소지를 없이 보낼 경우!! == 참조값없이 객체를 보낼 경우
null을 가리킬때 발생하는 예외 상황!!

* 접근 제어자
-> public, private 같은 접근 제어자를 제공한다. 이 제어자를 사용하면 해당 클래스를 외부에서 특정 필드나 메서드에 접근을 허용하거나 제안할 수 있음
** 접근 제어자 종류
-> private : 모든 외부 호출을 막는다.
-> default : 같은 패키지안에서 호출 허용
-> protected : 같은 패키지안에서 호출 허용, 패키지가 달라도 상속 관계의 호출은 허용
-> public : 모든 호출 허용

** 접근 제어자 사용 - 클레스 레벨
-> public, default만 사용 가능
-> public은 반드시 파일명과 이름이 같아야 한다.

* 캡슐화
-> 속성과 기능을 하나로 묶고, 외부에 꼭 필요한 기능만 노출 하고 나머지는 숨기는 것
-> 데이터(속성) + 기능(메서드)을 숨겨라
--> 데이터(속성) : 볼륨 예시를 생각해라 (100까지만 볼륨을 올릴 수 있다는 조건) => 거의 그래서 private으로 막아둔다.
--> 기능(메서드) : 필요한 기능만 오픈하고 나머지는 잠궈라 예를 들어 자동차를 운전하기 위해서 제공하는 복잡한 엔진 조절, 배기 등 우리가 다 모든 기능을 알 필요 없다.
 => 사용자 입장에서 꼭 필요한 기능만 외부로 노출

* 자바 메모리 구조
- 메서드 영역 = 붕아빵 틀 -> 클래스 정보를 보관한다
    -- 프로그램을 실행하는 필요한 공통 데이터를 관리한다
        - class 정보, static 영역, 런타임 상수 풀, 리터럴 문자가 있으면 공용 문자로 묶어서 관리(자바의 최적화 파트)
- 스택 영역 -> 실제 프로그램이 실행되는 파트
    -- 자바를 실행 할때 하나의 실행 스택이 생성된다. 각 스택 프레임은 지역 변수, 중간 연산 결과, 메서드 호출 정보 등이 포함된다.
- 힙 여역 = 붕어빵 -> 객체(인스턴스)가 생성되는 영역, 참고로 배열도 이 영역에 생성된다.

*스택과 규 자료 구조
- 넣는 곳과 빼는 곳이 다 같다 1->2->3 블록으로 쌓을 경우 뺄때는 3 -> 2-> 1 순서로 빠진다
Last in first out = 후입 선출 = 스택
- First in frist out = 선입 선출 = 큐

* 스택 영역
- 자바는 스택 영역을 사용해서 메서드 호출과 지역 변수(매개변수 포함)을 관리
- 메서드를 계속 호출하면 스택 프레임이 계속 쌓인다
- 지역 변수는 스택 영역에서 관리
- 스택 프레임이 종료되면 지역 변수도 함께 제거
- 스택 프레임이 모두 제거되면 프로그램도 종료

* static 변수
- static 키워드는 보통 멤버 변수와 메서드에 사용된다.

* final
- 변수에 final이 붙으면 더 이상 값을 변경할 수 없다라는 뜻
* 상수
- 변하지 않고, 항상 일정한 값을 갖는 수 (고정된 값)
- static final 키워드 사용
- 대문자 사용하고 구분은 _로 한다
- 필드를 직접 접근이 가능하다.

👉 static = “공유할거야 (클래스 소속)”
👉 final = “못 바꾸게 할거야 (불변)”
👉 static final = “공유 + 불변 = 상수”

* 상속과 메모리 구조
